# SQL 填写 Agent 需求文档

## 1. 项目概述

**项目名称**：SQL 填写 Agent
**项目类型**：Web / 桌面工具
**核心功能**：通过可视化界面导入数据库结构（元数据），用户通过勾选字段即可自动生成 SQL 查询语句，支持聚合统计与多表联查。
**目标用户**：需要快速编写 SQL 查询的非 DBA 技术人员、产品、运营、数据分析师。

---

## 2. 功能需求

### 2.1 数据库连接与元数据导入

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| F-001 | 支持连接 MySQL 数据库，填写 Host、Port、User、Password、Database 即可一键导入 | P0 |
| F-002 | 导入后展示完整的数据库 → 表 → 字段三层树形结构 | P0 |
| F-003 | 每个字段显示：字段名、字段类型、字段注释（名称） | P0 |
| F-004 | 支持切换当前操作的数据库 | P1 |

### 2.2 SQL 自动生成

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| F-005 | 用户勾选目标字段后，自动拼接 `SELECT field1, field2, ... FROM table` | P0 |
| F-006 | 自动保留字段注释作为别名（如 `SELECT username AS 用户名`） | P0 |
| F-007 | 支持 `WHERE` 条件筛选，提供字段名、运算符（=`!=``>` `<``LIKE``IN`）、值输入框 | P1 |
| F-008 | 支持 `ORDER BY` 升序/降序排列 | P1 |
| F-009 | 支持 `LIMIT` 条数限制 | P1 |

### 2.3 聚合统计功能

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| F-010 | 支持对数值字段进行：`SUM()` 求和、`AVG()` 平均值、`COUNT()` 计数、`MAX()` 最大值、`MIN()` 最小值 | P0 |
| F-011 | 支持 `GROUP BY` 分组，勾选分组字段后自动生成 | P0 |
| F-012 | 聚合模式下，非聚合字段必须加入 GROUP BY 或使用聚合函数 | P0 |

### 2.4 多表联查

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| F-013 | 支持选择多张表进行联查 | P0 |
| F-014 | 联查时，用户手动指定每对表之间的 `ON` 匹配字段（左表字段 = 右表字段） | P0 |
| F-015 | 支持 `INNER JOIN`、`LEFT JOIN`、`RIGHT JOIN` 三种类型可选 | P0 |
| F-016 | 联查字段从已选主表字段 + 被联表字段中分别选择 | P0 |

### 2.5 导出与复制

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| F-017 | 生成的 SQL 一键复制到剪贴板 | P0 |
| F-018 | 支持将当前查询配置保存为模板（JSON 格式），下次加载 | P1 |

---

## 3. 非功能需求

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| N-001 | 纯前端实现，不依赖后端服务，数据库元数据仅在客户端内存中处理 | P0 |
| N-002 | 响应式布局，兼容 1024px 以上屏幕 | P1 |
| N-003 | 支持深色/浅色主题切换 | P2 |

---

## 4. 技术方案（建议）

| 选项 | 方案 | 说明 |
|------|------|------|
| 前端 | React + TypeScript | 组件化开发，状态管理清晰 |
| 样式 | Tailwind CSS | 快速搭建响应式 UI |
| 数据库连接 | 前端直连 MySQL（需配置 CORS 代理）或提供 JSON 导入功能 | 出于安全考虑，建议提供 JSON 导入替代直连 |
| 状态管理 | React Context / Zustand | 轻量级状态共享 |
| SQL 编辑 | Monaco Editor | 高亮语法、格式化 |

> **备选方案**：若数据库不允许前端直连，可改为 **JSON 导入**模式——用户通过 mysqldump 或脚本导出元数据为 JSON 后导入本工具，完全离线运行。

## 5. 页面结构

```
┌─────────────────────────────────────────────────────────────┐
│  Header：Logo + 数据库选择 / 连接状态                         │
├──────────────┬──────────────────────────────────────────────┤
│              │  SQL 编辑区（Monaco Editor）                  │
│  数据结构树    │  ┌──────────────────────────────────────────┐│
│  - 数据库     │  │  SELECT ... FROM ... WHERE ...          ││
│    - 表1      │  │  GROUP BY ... HAVING ...                ││
│      - 字段1  │  │  ORDER BY ... LIMIT ...                 ││
│      - 字段2  │  └──────────────────────────────────────────┘│
│    - 表2      │                                              │
│  ...          │  ┌──────────────────────────────────────────┐│
│              │  │ [复制 SQL] [保存模板] [清空]                ││
├──────────────┴──┴──────────────────────────────────────────┤
│  配置面板（可折叠）：WHERE / GROUP BY / JOIN / ORDER BY / LIMIT│
└─────────────────────────────────────────────────────────────┘
```

## 7. 验收标准

- [ ] 可通过 JSON 导入数据库结构，树形展示正确
- [ ] 勾选字段后 SELECT 语句生成正确，注释作为别名
- [ ] 支持 SUM/AVG/COUNT/MAX/MIN 聚合
- [ ] GROUP BY 自动生成，非聚合字段自动加入 GROUP BY
- [ ] 两表 INNER JOIN / LEFT JOIN / RIGHT JOIN 可正确生成 ON 条件
- [ ] SQL 可一键复制
- [ ] 无后端，纯前端单文件可运行
