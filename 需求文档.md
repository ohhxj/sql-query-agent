# SQL 填写 Agent — 需求文档

## 1. 核心需求

### 1.1 元数据导入
- 支持导入数据库结构：库 → 表 → 字段 → 字段注释（中文名）
- 导入格式：JSON（用户通过工具脚本从数据库导出）
- 展示为树形结构：库 > 表 > 字段（含类型和注释）

### 1.2 表的分组管理
- 每个表除了归属库之外，支持打**项目标签**（可多个标签）
- 支持按标签快速筛选/过滤表
- 标签管理：增、删、改标签

### 1.3 功能页（SQL 生成与编辑）

#### 1.3.1 字段选择
- 从当前操作的表选择字段
- 支持选择**原始字段**（直接显示字段名）
- 支持选择**统计后字段**（SUM/AVG/COUNT/MAX/MIN）
- 联表后，支持从被联的表也按上述方式选择字段

#### 1.3.2 排序
- 支持为每个已选字段选择排序方式（升序/降序/不排序）

#### 1.3.3 生成
- 点击生成按钮，根据已选字段、联表配置、排序配置生成 SQL 语句

#### 1.3.4 手动编辑
- 生成后的 SQL 显示在可编辑区域
- 支持用户手动修改 SQL 内容

#### 1.3.5 AI 校验与风险评估
- 对已生成/编辑的 SQL 进行 AI 校验
- 风险评估包括但不限于：全表扫描风险、缺少索引警告、跨库查询警告、SQL 注入风险等
- 提供风险等级提示（高/中/低）

## 2. 数据结构（JSON 导入格式）

```json
{
  "databases": [
    {
      "name": "shop_db",
      "comment": "商城数据库",
      "tables": [
        {
          "name": "orders",
          "comment": "订单表",
          "tags": ["电商", "核心"],
          "fields": [
            { "name": "id", "type": "int", "comment": "订单ID" },
            { "name": "user_id", "type": "int", "comment": "用户ID" },
            { "name": "amount", "type": "decimal", "comment": "订单金额" },
            { "name": "created_at", "type": "datetime", "comment": "下单时间" }
          ]
        },
        {
          "name": "users",
          "comment": "用户表",
          "tags": ["用户", "核心"],
          "fields": [
            { "name": "id", "type": "int", "comment": "用户ID" },
            { "name": "username", "type": "varchar", "comment": "用户名" }
          ]
        }
      ]
    }
  ]
}
```

## 3. 页面结构

```
┌──────────────────────────────────────────────────────────┐
│  Header：SQL Agent 标题 + 导入JSON按钮 + 标签筛选输入框    │
├──────────────┬───────────────────────────────────────────┤
│              │  已选字段区（当前表字段 + 联表字段 + 统计）  │
│  库-表-字段   │  ┌───────────────────────────────────────┐ │
│  树形结构     │  │ orders.id ✓ | users.username ✓       │ │
│  （左侧可     │  │ SUM(amount) AS 订单总额 ✓              │ │
│  折叠）       │  └───────────────────────────────────────┘ │
│              │                                            │
│  项目标签     │  排序配置区（字段 → 升序/降序/不排序）       │
│  筛选区       │                                            │
│  （横向标签   │  联表配置区（选择联表 + 匹配字段 + JOIN类型） │
│   按钮组）    │                                            │
│              │  ┌───────────────────────────────────────┐ │
│              │  │  SQL 编辑区（Monaco Editor）           │ │
│              │  │  SELECT ... FROM ...                  │ │
│              │  │  [生成] [AI校验] [风险评估] [复制]      │ │
│              │  └───────────────────────────────────────┘ │
└──────────────┴───────────────────────────────────────────┘
```

## 4. 验收标准

- [ ] 可导入 JSON，库-表-字段树形展示正确
- [ ] 表可打标签，支持按标签筛选
- [ ] 可从表选择原始字段和统计后字段
- [ ] 联表后可从被联表选择原始字段和统计后字段
- [ ] 字段可选排序方式（升序/降序/不排序）
- [ ] 点击生成生成正确 SQL
- [ ] SQL 可手动编辑
- [ ] AI 校验功能存在（校验语法/逻辑）
- [ ] 风险评估功能存在（提示全表扫描等风险）
- [ ] SQL 可一键复制

## 5. 技术栈（建议）

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | React + TypeScript | 组件化 |
| 样式 | Tailwind CSS | 响应式 UI |
| SQL 编辑器 | Monaco Editor | 高亮+编辑 |
| 状态管理 | Zustand | 轻量 |
| AI 校验 | 调用 AI 接口（Claude/GPT） | 发送 SQL 返回校验结果 |
| 纯前端 | 不依赖后端 | 元数据存内存 |
